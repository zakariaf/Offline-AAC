Fix four things. (1) Drop "unrecoverable": drift_schemas/*.json are derived artifacts. `schema dump` reads the schema from the Dart source (and per /migrations/exports/ also accepts a sqlite3 database file as its first argument), so a missed version is regenerated by checking out that version's commit/tag and re-running the dump. The JSON dumps are DERIVED artifacts, not the source of truth. schema dump reads the database definition out of the Dart source file, so checking out the git commit/tag for version N and re-running the dump regenerates drift_schema_vN.json. Independently, /migrations/exports/ states "drift_dev schema dump recognizes a sqlite3 database file as its first argument" — a historical schema can also be dumped from an actual on-disk database file at that version. Recovery is inconvenient, not impossible; what would make it unrecoverable is losing the code history, not gitignoring drift_schemas/.
2. 3. Treating a generated dir as structural is the cargo-cult tell here.
